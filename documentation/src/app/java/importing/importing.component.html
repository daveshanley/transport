Importing Transport into your Spring Application

The Java version of Transport can be consumed via gradle, or maven.

What does the Transport depend on?

Magic doesn't come for free, so Transport heavily depends on RxJava, Spring, Jackson, Apache Commons and a few others.

Configuring Gradle

Add a dependency to the desired Transport jar version and its dependencies. 

dependencies {
    compile "com.vmware.transport:vmw-transport:1.0.0"
    compile "org.springframework.boot:spring-boot-starter-web:1.5.18.RELEASE"
    compile "org.springframework.boot:spring-boot-starter-websocket:1.5.18.RELEASE"
    compile "org.springframework.security:spring-security-web:4.2.2.RELEASE"
    compile "org.springframework.security:spring-security-config:4.2.2.RELEASE"
    compile "io.reactivex.rxjava2:rxjava:2.1.9"
    compile "com.fasterxml.jackson.core:jackson-databind:2.9.4"
    compile "com.fasterxml.jackson.module:jackson-module-jsonSchema:2.9.4"
    compile "org.slf4j:slf4j-api:1.7.25"
    compile "ch.qos.logback:logback-classic:1.2.3"
    compile "com.google.code.gson:gson:2.8.1"
    compile "joda-time:joda-time:2.9.9"
    compile "org.apache.httpcomponents:httpclient:4.5.6"
    compile "org.apache.commons:commons-lang3:3.8.1"
}
